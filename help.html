Overview

What does this actually do?

Imagine a real camera person holding a camera and zooming in to follow your mouse as you work on screen. That's exactly what this script does - but it's automatic, inside OBS, every single frame.

🖱️

Mouse Tracking

The core feature

The script watches where your mouse cursor is every frame. It zooms and pans your screen recording so your cursor is always near the centre of the frame. Move your mouse left - the camera follows. Move to the bottom - it follows there too. Smoothly, like a real camera operator.

✂️

How it works inside OBS

No magic, just maths

It adds a hidden crop filter to your capture source. Every frame, it trims the edges of your screen so that a zoomed-in window is what remains. OBS stretches that window to fill your canvas - that's the zoom. The crop shifts to follow your mouse - that's the pan.

Nothing is recorded twice. No post-processing needed. What you see in the OBS preview is exactly what gets recorded or streamed.

📱

Perfect for vertical content

YouTube Shorts · Instagram Reels · TikTok

You might record your screen for YouTube (wide, 16:9) and also want the same session as a vertical (9:16) video for Reels. This script handles the vertical crop automatically so you record both formats in one take - with the camera intelligently following your mouse so viewers always see the right part of your screen.

💡

In one sentence: You record your screen normally. When this script is on, OBS zooms in and pans around to follow your mouse - making your recording look cinematic and easy to follow without you doing anything extra.

Installation

Installing the script

You only do this once. It takes about two minutes.

1

Download the files

Go to the GitHub releases page and download the latest .zip file. Unzip it anywhere - your Desktop is fine.

2

Run the installer

Run ProAutoZoomUltimate_v1.0_Setup.exe - it automatically installs the script into your OBS scripts folder, then shows you the next steps. No manual file copying needed.

⚠️

Windows may ask "Do you want to allow this app to make changes?" Click Yes. The installer only copies one file to your OBS scripts folder. You can open it in Notepad to read exactly what it does.

3

Open OBS and go to Tools → Scripts

In OBS, click the Tools menu at the top, then click Scripts. A window opens showing all installed scripts.

4

Add the script

Click the + button (bottom left of the Scripts window). The scripts folder that opened in step 2 - select Pro_auto_zoom_ultimate_v1.0.lua and click Open. The script settings appear on the right immediately.

5

Done - the script loads automatically from now on ✓

Every time OBS opens, this script loads automatically. No need to add it again.

ℹ️

Manual install (no installer): Copy Pro_auto_zoom_ultimate_v1.0.lua to any location of your choice and then follow steps 3–4 above.

Quick Start

Getting tracking working in 5 steps

After installing, tell the script which source it's zooming and which monitor to watch.

1

Pick your Capture Source

In the script settings (right side of the Scripts window), find the Capture Source dropdown. Click it and select your screen/monitor/game capture source - the one you want the zoom to work on.

💡

Not sure which source? Look at your OBS scene. The one named something like "Display Capture" or "Monitor Capture" is usually it.

2

Pick your Detected Monitor

Under Monitor & Display Settings, click the Detected Monitor dropdown. It lists every monitor connected to your PC - pick the one your capture source is recording from.

⚠️

This step is required. The script needs to know exactly where your monitor sits in Windows to map your cursor position correctly. Without it, the hotkey won't activate and you'll see a warning in the log.

3

Pick a Canvas Layout

Under Source & Layout, the Canvas Layout dropdown tells the script how your canvas is arranged. For most setups: Full Screen Zoom. For vertical content with a webcam on top: Split: Webcam Top, Screen Bottom. See the Layouts section for pictures of each.

4

Assign your hotkey

In OBS, go to Settings → Hotkeys. Search for "Pro Autozoom Ultimate". Assign a key to Toggle Camera - this is what turns the zoom on and off. Something like F9 works well.

5

Press your hotkey and move your mouse!

Press the Toggle Camera hotkey. Move your mouse - watch the OBS preview zoom in and follow your cursor in real time. Press the hotkey again to turn it off and return to the full-screen view.

Settings Reference

Every single setting explained

Here's what every control in the script does - in plain language. Settings are shown in the same order they appear in OBS.

🎥 Source & Layout

Capture Sourcedropdown
Which OBS source should the zoom apply to? Pick your monitor, window, or game capture source. This is the source that gets cropped and zoomed to follow your mouse. If your source doesn't appear, make sure it's a Monitor Capture, Window Capture, or Game Capture type - not a webcam, image, or browser source.
Canvas Layoutdropdown
How is your OBS canvas arranged? This tells the script where the screen-share lives on your canvas. Choose Full Screen Zoom if your entire canvas is the screen recording. Choose a Split option if a webcam takes up part of the canvas. See Layouts below for a visual guide to each option.
Webcam Height (px)numberonly for split layouts
How tall is your webcam area, in pixels? Only appears when a Split layout is chosen. If your webcam takes up 608 pixels at the top of your canvas, type 608. The script then uses the remaining height for the screen-share zoom. Find this number by right-clicking your webcam source in OBS → Edit Transform → look at the Size → Height value.
Enable Manual Canvas Overridecheckboxdefault: off
Only turn this on if you use a separate vertical canvas plugin like Aitum Vertical. Normally the script reads your canvas size from OBS automatically - leave this off and everything just works. If you use Aitum Vertical, OBS's scripting API can't report that separate canvas's size, so the script would read your main 16:9 canvas and produce a squashed image. Tick this box and enter your actual canvas width and height to fix it.
Vertical Canvas Width (px)numberonly with override on
The full width of your vertical output canvas. For a standard vertical canvas: 1080.
Vertical Canvas Height (px)numberonly with override on
The full height of your vertical output canvas. For a standard vertical canvas: 1920.

🖥️ Monitor & Display Settings

Detected Monitordropdown⚠️ required
Which physical monitor is being captured? The script reads your mouse's real coordinates from Windows and maps them onto your recorded image. To do that correctly it needs to know exactly where your monitor sits in your Windows display layout. Pick the monitor that matches your capture source. The format is:

Monitor 1 - 1920×1080 @ (0,0) (Primary)

That means: Monitor 1, 1920px wide, 1080px tall, at position 0,0 in Windows, and it's your main display.
Enable Manual Monitor Overridecheckboxdefault: off
Advanced: type in your own monitor dimensions instead of using the auto-detected ones. Leave this off unless the auto-detected values are wrong. When ticked, four fields appear for Width, Height, X Offset, and Y Offset - these let you correct the geometry for unusual multi-monitor setups where auto-detect fails.

🎬 Camera Engine

Enable Camera Trackingcheckboxdefault: on
The master on/off switch. When unticked, all hotkeys do nothing and tracking is completely disabled. Useful if you want to keep the script loaded but temporarily paused without having to remove it.
Base Zoom Factorslider · 1.0–5.0default: 2.0
How much does the camera zoom in? Think of it like a magnifying glass strength.

1.0 = no zoom (full screen visible - tracking on but no magnification)
2.0 = 2× zoom - you see half the screen. A great starting point.
3.0 = 3× zoom - you see a third of the screen. Good for detailed work.
5.0 = 5× zoom - very close up. Can feel overwhelming if you move fast.

Higher zoom = less of your screen is visible at once, but what you do see is much larger. Start at 2.0 and adjust.
Punch Zoomslider · 1.0–10.0default: 4.0
How much extra zoom when you hold the Punch Zoom hotkey? While you hold the Punch key, the camera zooms in to this level instead of Base Zoom. Release and it eases back to normal. Use it to draw attention to something specific - a single button, a line of code, a field in a form. Should be set higher than your Base Zoom. Default is 4.0 (4× while held).
Smoothnessslider · 0.01–0.50default: 0.12
How quickly does the camera glide to follow your mouse? This is a speed multiplier applied every frame.

Lower number = slower and smoother. Higher number = faster and snappier.

0.04–0.08: Very cinematic, lazy-camera feel - great for presentations.
0.12: Default - smooth and responsive.
0.25+: Snappy, almost instant. Can look jittery if you move your mouse quickly.

Important: lower smoothness often looks more professional on video, not worse. Don't be afraid to go low.
Deadzone (%)slider · 0–40default: 15
A "calm zone" in the middle where small mouse movements don't move the camera. Imagine a box in the centre of your screen. As long as your mouse stays inside that box, the camera stays still. The moment your cursor leaves the box, the camera starts following. This stops the camera from jiggling every time you make small accidental movements.

0%: No calm zone - camera follows every tiny twitch (can look shaky).
15%: Default - a comfortable middle ground.
30%+: Very stable camera that only moves when you go far from centre.
Auto-Return to Center when Idlecheckboxdefault: off
Should the camera drift back to the middle of the screen when you stop moving your mouse? When ticked, if your mouse stays still for the Idle Timeout duration, the camera slowly glides back to show the centre of your screen. Good for presentations where you want a centred default view when you're not actively pointing at anything.
Idle Timeout (s)slider · 1.0–10.0default: 3.0
How many seconds of mouse stillness before the camera returns to centre? Only relevant when Auto-Return to Center is ticked. 3.0 means: if your mouse doesn't move for 3 seconds, the camera starts gliding back to centre.
Debug Modecheckboxdefault: off
Turns on detailed logging - for troubleshooting only. When ticked, the script writes a line to the OBS log every time the camera moves, a setting changes, or a hotkey is pressed. View the log at Tools → Scripts → the log icon. Turn this off during normal use - it generates a lot of text and slightly increases CPU usage. Only turn it on if something isn't working and you want to diagnose why.
↺ Reset Camera Engine to Defaultsbutton
Resets all Camera Engine sliders and checkboxes to their factory values: zoom 2.0, punch zoom 4.0, smoothness 0.12, deadzone 15%, auto-center off, idle timeout 3.0s, debug mode off. Click this if you've been experimenting and want to start fresh.

🎯 Mouse Indicator

ℹ️

The Mouse Indicator is an optional glowing ring that appears around your cursor in the recording. Viewers can see exactly where your mouse is at a glance. The script creates and controls this as an OBS source automatically - you don't set anything up manually.

Visibilitydropdowndefault: Off
When should the ring appear?

Off - no indicator. (The color/opacity/size controls disappear since they'd do nothing.)
Always On - the ring is always visible while tracking is active.
Hotkey Triggered - the ring only appears when you press the Toggle Pointer hotkey. Good for showing it only during key moments.
Ring Colorcolour pickerdefault: yellow
What colour should the ring be? Click the colour box to open a colour picker. Yellow is the default because it's visible on most backgrounds. Pick anything that contrasts with your typical screen content - bright colours like yellow, cyan, or white work best.
Ring Opacity (%)slider · 10–100default: 70%
How transparent should the ring be? 100% = fully solid. 10% = barely visible. 70% (default) is a good balance - visible but not distracting.
Ring Size (px)slider · 20–300default: 72px
How big should the ring be? Measured in pixels on the output. At 2× zoom, 72px feels natural. If you use a higher zoom factor, a smaller ring (40–50px) looks better. At lower zoom, a bigger ring (90–120px) is more visible.
↺ Reset Mouse Indicator to Defaultsbutton
Resets all Mouse Indicator settings: Visibility → Off, Ring Color → yellow, Opacity → 70%, Size → 72px.
Hotkeys

Keyboard shortcuts

There are four hotkeys. Assign them in OBS under Settings → Hotkeys - search for "Pro Autozoom Ultimate".

🎬 Toggle Camera

The main switch. Press once to turn tracking ON. Press again to turn it OFF and return to the normal full-screen view instantly. This is the hotkey you'll use most - assign it somewhere easy like F9.

🔍 Hold for Detail Zoom (Punch)

Hold this key to zoom in tighter (to your Punch Zoom level). Release and it smoothly eases back to Base Zoom. Perfect for highlighting a specific button, line of code, or cell in a spreadsheet. It's a hold key - not a toggle.

⏸️ Hold to Freeze Camera

Hold this key to lock the camera - it stops following your mouse. Release and tracking resumes. Use it when you're about to move your mouse far away (like scrolling a page) but want viewers to keep looking at what's currently on screen.

🎯 Toggle Pointer

Shows or hides the Mouse Indicator ring. Only works when Visibility is set to Hotkey Triggered. Press to show the ring, press again to hide it. Great for drawing attention to your cursor at a specific moment.

💡

Hotkeys not saving? After assigning keys in OBS Settings, make sure to click Apply or OK. OBS only saves hotkeys when you confirm the Settings dialog - just assigning and closing isn't enough.

Vertical Video

Recording vertical content (9:16)

For people producing Reels, Shorts, or TikTok - especially if you use Aitum Vertical or a similar plugin to create a separate vertical canvas.

📐

Why vertical needs special treatment

Your monitor is wide (1920×1080). Your vertical canvas is tall (1080×1920). When the script zooms into a wide monitor to fill a tall canvas, it needs to cut out a slice that has the correct tall proportions. If it gets the canvas size wrong, the slice is the wrong shape and the image looks squashed horizontally.

The script calculates the correct shape automatically - but it needs to know the exact size of your output canvas to do so.

🔄

Using Aitum Vertical (or a similar plugin)

Aitum Vertical creates a completely separate vertical canvas inside OBS. Unfortunately OBS's scripting API can only see your main canvas - it can't see Aitum's separate canvas. So if the script tries to auto-detect the size, it reads your 16:9 main canvas and gets the wrong answer.

The fix - four steps:

  1. Tick Enable Manual Canvas Override in Source & Layout.
  2. Set Vertical Canvas Width to 1080.
  3. Set Vertical Canvas Height to 1920.
  4. Set Webcam Height to however many pixels your webcam takes up (e.g. 608). Find this by right-clicking your webcam source → Edit Transform → Size → Height.
🖼️

How to set up the screen-share source transform in OBS

Right-click your screen-share source in the vertical scene → Edit Transform. Set it like this:

  • Position X: 0   Position Y: [your webcam height, e.g. 608]
  • Size Width: 1080   Size Height: [1920 − webcam height, e.g. 1312]
  • Bounds type: Fit   Bounds Width: 1080   Bounds Height: 1312
  • Crop (at the bottom of the Transform dialog): all 0 - the script's filter handles all cropping, don't set this manually.
💡

Check that the script's crop filter (CoreAutoZoomUltimate_Crop, found in Filters on your source) has Relative: ticked. This makes the crop reduce the source's actual output size, which is what allows Fit bounds to work correctly without distortion.

🩺

Diagnosing the squash with the canvas diagnostic

Not sure if you need the manual override? Turn on Debug Mode, make any settings change, then open Tools → Scripts → the log icon. Look for a line like:

CANVAS DIAG: OBS API base=1920x1080 | script target=1080x1920 (webcam 608px, override=true)

If OBS API base shows 1920×1080 while you're producing vertical content, the auto-read is seeing the wrong canvas - enable the manual override. If it shows 1080×1920, auto-read is working and you don't need the override.

Also check the OutRatio and TargetRatio values in the CROP log lines. If they match (e.g. both 0.823), the crop shape is correct and the image should look undistorted.

Troubleshooting

Something's not working?

Click any problem to expand the solution.

The hotkey does nothing when I press it
Check these in order:
  1. No monitor selected. Go to Monitor & Display Settings → Detected Monitor → pick your monitor. This is required.
  2. "Enable Camera Tracking" is unticked. Go to Camera Engine → make sure it's ticked.
  3. Hotkey wasn't saved. Go to OBS Settings → Hotkeys → search "Pro Autozoom Ultimate" → assign the key → click Apply → click OK.
The image looks squashed or stretched horizontally
The script calculated the crop shape using the wrong canvas dimensions. Almost always happens with Aitum Vertical or similar plugins.

Fix: Go to Source & Layout → tick Enable Manual Canvas Override → set Width: 1080, Height: 1920 (or your actual vertical canvas size). Set Webcam Height to your actual webcam pixel height.

Turn on Debug Mode and check the CANVAS DIAG line (see the Vertical Video section above for details on reading it).
The camera follows the wrong area / tracks the wrong monitor
Your Detected Monitor is set to the wrong display.

Go to Monitor & Display Settings → Detected Monitor → pick the monitor your capture source is actually recording. On a two-monitor setup, try the other one.

With Debug Mode on, the CROP log lines show your mouse coordinates and whether the script thinks the cursor is on the right screen (OnScreen: true or OnScreen: false). If you see false while your mouse is clearly on the recording monitor, the wrong monitor is selected.
The camera movement looks jittery or shaky
Try these adjustments in Camera Engine:
  • Lower the Smoothness value (e.g. from 0.12 to 0.06). Lower = smoother, slower camera movement - this is what you want.
  • Increase the Deadzone (e.g. from 15 to 25). A larger calm zone stops the camera reacting to small accidental mouse movements.
  • Lower the Base Zoom Factor. Higher zoom amplifies every pan - reducing it makes movement feel smaller and smoother.
The script log keeps filling up with identical lines even when I'm not moving
This was a known bug in earlier versions. It's fixed in v1.0 - the script now detects when the camera has settled and goes silent.

If you're on v1.0 and still see this, make sure Debug Mode is turned off. Most tick logs only appear with Debug Mode on.
I see a filter called "CoreAutoZoomUltimate_Crop" on my source - is that normal?
Yes, completely normal. The script creates this filter automatically on your selected source. It adjusts the crop values every frame to move the camera. Do not delete it while the script is running. When the script is removed or disabled, it cleans up the filter automatically.
The Mouse Indicator ring isn't showing up
  1. Make sure Visibility in Mouse Indicator is not set to Off.
  2. If it's Hotkey Triggered, you need to press the Toggle Pointer hotkey to show it.
  3. Make sure tracking is active (press Toggle Camera to start it).
  4. The ring only shows when your cursor is on the monitored screen. It hides automatically when you move to a different monitor.
How do I uninstall the script?
Easy: Go to Windows Settings → Apps, search for Pro Autozoom Ultimate, and click Uninstall. Or run the uninstaller from where you installed.

Manual: In OBS → Tools → Scripts → click the script in the list → click the − button. Then delete the .lua file from your scripts folder to clean up completely.

When the script is removed, it automatically restores your capture source to its normal uncropped view - nothing is left broken.
Good to know

Known limitations

🪟

Windows only

The script reads your mouse position and monitor geometry using Windows-specific system calls. OBS itself runs on Mac and Linux, but this script does not work on those platforms.

1️⃣

One source at a time

The script tracks one capture source with one set of settings. The crop filter and indicator use fixed internal names, so running two instances of the script can cause conflicts. Multi-source support is a possible future improvement.

🖱️

Works best with full-screen captures

The script maps your cursor to a fixed monitor region. If you use a Window Capture and the captured window is not fullscreen - or if you move/resize it mid-recording - the cursor mapping can drift. Works most reliably with a Monitor Capture or a fullscreen game/app.

🚧

Picture-in-Picture (PiP) layout not yet available

A Picture-in-Picture canvas layout (where the script avoids the corners used by a PiP webcam overlay) is planned but not yet implemented. It's intentionally not shown in the Canvas Layout dropdown to avoid confusion.
